# Contributing to FST344 Sequence Optimiser

Thanks for your interest in contributing. This project sits at the intersection of computational biology and gene therapy -- contributions from both wet-lab scientists and bioinformaticians are valuable.

## How to contribute

1. **Fork** the repository
2. **Create a branch** for your changes (`git checkout -b feature/your-feature`)
3. **Make your changes** and commit with clear messages
4. **Open a pull request** with a description of what you changed and why

## What we're looking for

### High-impact contributions

- **Experimental validation data** -- transfection results, expression comparisons, or in vivo data using the optimised sequences
- **Alternative optimisation strategies** -- different codon tables, tissue-specific optimisation, or machine learning approaches
- **Structural analysis** -- molecular dynamics simulations, binding affinity predictions, or additional AlphaFold runs
- **Signal peptide engineering** -- improved secretion efficiency for muscle cell expression

### Always welcome

- Bug fixes in the Python tools
- Documentation improvements
- Additional sequence analysis scripts
- Test coverage for the optimisation algorithms
- Corrections to the literature review

## Guidelines

- **Keep it scientific.** Changes to optimisation parameters should be justified with references or data.
- **Preserve protein identity.** Any sequence changes must encode the identical 344-amino-acid protein.
- **No unnecessary dependencies.** The Python tools are intentionally dependency-free (stdlib only). New tools can use external packages, but document them.
- **Write clear commit messages.** Describe what changed and why.

## Reporting issues

Open an issue for:
- Bugs in the optimisation scripts
- Incorrect biological assumptions
- Missing analysis or documentation
- Feature requests

## Code of conduct

Be professional. Be constructive. Focus on the science.
